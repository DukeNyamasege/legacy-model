# Test 2 Render Deployment

The Render Blueprint creates a FastAPI web service, one continuous worker, and
PostgreSQL. The worker lease prevents a second healthy worker from trading the
same Test 2 account scope.

## Deploy

1. Push this branch to a private Git repository.
2. In Render, create a Blueprint from `render.yaml`.
3. Supply the prompted `DERIV_APP_ID` and `DERIV_TOKEN` values.
4. Supply `DERIV_TOKEN_ENCRYPTION_KEY` so dashboard-submitted copier tokens are
   encrypted at rest.
5. Keep every trading environment value on `demo` for Test 2 unless you
   intentionally switch the runtime mode to `real` in the dashboard.
6. After deployment, open `/health/live`, `/health/ready`, then `/status`.
7. Enter the web service's `CONTROL_API_KEY` in the dashboard control-key field.
8. Watch worker logs for `TRADER_LOCK_ACQUIRED`, WebSocket connections, and
   signal decisions.

Render currently accepts `runtime: docker`, `type: worker`,
`dockerCommand`, `preDeployCommand`, and database
`property: connectionString`. The included `basic-256mb` PostgreSQL plan is a
paid Render resource; review the Blueprint cost before confirming it.

## Required Environment Variables

Web service:

- `DATABASE_URL`: injected from Render PostgreSQL.
- `DERIV_APP_ID`: current developers.deriv.com application ID.
- `CONTROL_API_KEY`: generated by Render.
- `DERIV_TOKEN_ENCRYPTION_KEY`: Fernet key for dashboard-managed copier tokens.
- `TRADING_MODE=demo`
- `DERIV_ENVIRONMENT=demo`
- `ALLOW_REAL_TRADING=false`
- `TEST_RUN_ID=bin22001`
- `DEPLOYMENT_ID=render-api`

Worker:

- `DATABASE_URL`: injected from the same PostgreSQL instance.
- `DERIV_APP_ID`: current developers.deriv.com application ID.
- `DERIV_TOKEN`: PAT with `trade` scope.
- `DERIV_TOKEN_ENCRYPTION_KEY`: same Fernet key used by the web service.
- `DERIV_ENVIRONMENT=demo`
- `TRADING_MODE=demo`
- `ALLOW_REAL_TRADING=false`
- `TEST_RUN_ID=bin22001`
- `DEPLOYMENT_ID=render-worker`

Do not add a token to `config.yaml`, `render.yaml`, logs, or exports. A real
account additionally requires the exact production acknowledgement documented in
the root README; changing only the account token cannot enable real purchases.

## Operations

Pause:

```bash
curl -X POST -H "X-API-Key: $CONTROL_API_KEY" https://YOUR_HOST/control/pause
```

Resume:

```bash
curl -X POST -H "X-API-Key: $CONTROL_API_KEY" https://YOUR_HOST/control/resume
```

Emergency stop:

```bash
curl -X POST -H "X-API-Key: $CONTROL_API_KEY" https://YOUR_HOST/control/emergency-stop
```
